iceberg-decentralized

A decentralized analytics platform — Trino + Apache Iceberg + IPFS + Zenoh, wired by a Go bridge service. Deploy on K3s with a single Helm command.

Concept

Traditional analytics stacks assume a single source of truth. This project doesn't.

RoleTechnologyWhy
Query engine Trino 435 Distributed SQL, Iceberg-native, federation-ready
Table format Apache Iceberg 1.4 ACID, time-travel, schema evolution, partition pruning
Content-addressed storage IPFS / Kubo 0.26 Data files referenced by CID — location-independent
Peer messaging Zenoh 0.11 Sub-ms pub/sub, works across NAT, edge-to-cloud
The glue zenoh-iceberg-bridge (Go) Publishes local partitions to Zenoh; pulls & verifies remote ones
System Architecture

System overview — 7 pods, 19 Kubernetes resources, one Helm install

Query Execution Flow

Query execution sequence — SQL → plan → splits → IPFS P2P fetch → merged result

Bridge Sync Loop

Bridge runs every 30s: publish local → pull remote → verify CID → register valid partitions

Quick Start
1
Clone & build the bridge image
git clone https://github.com/optimusdb/iceberg-decentralized.git
cd iceberg-decentralized/bridge
docker build -t optimusdb/zenoh-iceberg-bridge:0.1.0 .
# K3s: import directly without a registry
docker save optimusdb/zenoh-iceberg-bridge:0.1.0 | k3s ctr images import -
2
Install with Helm
helm install iceberg-decentralized . \
--namespace iceberg-decentralized \
--create-namespace \
--wait
3
Run your first decentralized query
trino --server http://<node-ip>:30080

-- Create a schema
CREATE SCHEMA iceberg.energy WITH (location = '/iceberg-warehouse/energy');

-- Create a partitioned table
CREATE TABLE iceberg.energy.readings (
ts TIMESTAMP, node_id VARCHAR, value DOUBLE
) WITH (format = 'PARQUET', partitioning = ARRAY['day(ts)']);

-- Query across nodes transparently
SELECT * FROM iceberg.energy.readings WHERE ts > TIMESTAMP '2026-01-01';
Note
Remote partitions from peer nodes appear automatically within 30 seconds of the bridge's first sync cycle — no manual registration needed.
